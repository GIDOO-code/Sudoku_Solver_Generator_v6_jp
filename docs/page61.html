POM分析 の結果を整理します：
  • それぞれのパターン内のセルは連結している。
  • パターン内のどれかのセルが候補数字が確定なら、連結する全てのセルは確定である。
  • パターン内のどれかのセルで候補数字が否定なら、連結する全てのセルでその候補数字は否定である。
なお、この分析は候補数字ごとの結果であり、候補数字間の相互関係は考慮されていません。



 POM アルゴリズム

POM分析の結果より、次のようなアルゴリズムを組み立てることができます。

  • 単一数字のアルゴリズム(Rule-1)
      • 候補数字#n 毎に
    • 全てのパターンに共通するセル・候補数字は確定である。 => All Patterns
    • 全てのパターンに含まれないセル・候補数字は除外できる。=> No Patterns
    • パターンが１種類のみの場合は、”全パターンに共通する”ので セル・候補数字は確定である。。=> single pattern

  • 候補数字の相互作用を含むアルゴリズム(Rule-2)
      • 相互作用による不成立パターンを除外
    • 有効パターンに共通するセル・候補数字は確定である。 =>PMI All Patterns
    • 有効パターンに含まれないセル・候補数字は除外できる。=>PMI No Pattern

 Pattern Overlay Interactions

候補数字の相互作用に基づくアルゴリズム（ルール2）は以下のとおりです。
候補数字#m のパターンA を真と仮定したとき(#m-A)、パターンA のセルは 候補数字#m が占有し、他の候補数字は排除されます。
この状態でPOM分析を行うと、他の候補数字のパターンがないことがあります。これは、#m-Aを真とする仮定が誤りなことを示します。
従って、候補数字#mのパターンA(#m-A)は除外されます。
Pattern Overlay Interaction(POI) は、候補数字の相互作用分を除外した状態での、POM分析です。

 Pattern Overlay Method の証明

排中律による証明：
POMは、候補数字の配置可能なパターンを網羅的に扱う解析アルゴリズムです。
これに対し、既存のアルゴリズムは、候補数字の配置を否定的に導きます。これは、その候補数字の配置が矛盾するからです。
POMのパターンは、矛盾のない配置のみです。すなわち、既存のアルゴリズムの矛盾する配置はPOMには含まれません。

上位互換：
単一数字を扱う 例えば ER、Skyscraper、Fish などのアルゴリズムに対し、POM Rule-1( no-pattern, all-pattern)は上位互換です。
複数数字を扱う 例えば XY-wing、Chain、SueDeCoq などのアルゴリズムには、候補数字の相互作用を分析する POI Rule-2(POI:Pattern Overlay interactions)が対応します。
