POM分析 の結果を整理します：
    候補数字 #n について、
  • それぞれのパターン内のセルは連結している。
  • パターンーが2種類のとき、パターンの一方は真であり、他方は偽である
  • パターンーが3種類以上のときは、1パターンのみが真であり、他のパターンは真と偽が混在する。
なお、この分析は候補数字ごとの結果であり、候補数字間の相互関係は考慮されていない。



 POM アルゴリズム

POM分析の結果をもとに、次のようなアルゴリズムを組み立てることができます。

  • 単一候補数字のアルゴリズム(Rule-1)
      • 候補数字#n 毎に
    • 全てのパターンに共通するセル・候補数字は真である。 => All Patterns
    • 全てのパターンに含まれないセル・候補数字は偽であり、除外できる。=> No Patterns
    • パターンが１種類のみの場合は、”全パターンに共通する”ので セル・候補数字は真である。
      ”全パターンに含まれない” セル・候補数字は偽である。        => All Patterns, single pattern

  • 候補数字の相互作用に基づくアルゴリズム(Rule-2)

  • POM と Nice-Loop の組み合わせアルゴリズム(Rule-3)

 POM - Interactions

候補数字の相互作用 に基づくアルゴリズム は以下のとおりです。

  • 候補数字#m のパターンX を真 と仮定する。
  • パターンX のセルは 候補数字#m が占有し、他の候補数字は排除される。
  • この状態でPOM分析を行う。
  • 他の候補数字のパターンが全くないことがある。この結果は認められない。
  • 従って、最初の仮定 候補数字#mのパターンX は偽であり、候補数字#m のパターンリストから X は除外される。
Pattern Overlay Method - Interaction(POM-I) は、パターンの絞り込みに高い効果をもつアルゴリズムです。

 POM - Interactions Level-2

候補数字の相互作用 に基づくアルゴリズム は、2つの候補数字の相互作用 のアルゴリズムへと、自然に拡張できます。

    相互作用 Level 2 アルゴリズム
  • 候補数字#m のパターンX1 と 候補数字#n のパターンX2> を真と仮定する。
  • 仮定のセル・数字が 占有すると、他の候補数字は排除される。
  • この状態でPOM分析を行う。
  • #m、#n 以外の候補数字 #x のパターンが求まらないことがある。この結果は認められない。
  • パターンの全組合わせのPOM分析結果を、次図のようにまとめる。
  • 列の全セルでパターンが求まらないなら、その列のパターンは否定であり、パターンリストから除外される。行についても同様である。
  • 相互作用分析の結果パターンが絞り込まれると、All-pattern、No-patternの判定が可能になる。


POM - Interaction Level-2 は、比較的少ないコストで高い解析力のアルゴリズムです。

 Pattern Overlay Method Rule3 - POM NiceLoop

ROM分析とNice-Loopを組み合わせて不整合を検出する アルゴリズムです。

  • それぞれのパターンを開始点として、NiceLoopの経路探索を行う。
  • 真のパターンを開始点としたときは、矛盾は検出されない。
  • 偽のパターンを開始点としたときは、矛盾を検出する。このパターンはセルの候補数字リストから除外する。
    (NiceLoopのループ探索では、"探索中に経路の矛盾：真偽の両方を導く" ことがある。
    この場合は探索を中断して、矛盾を検出したことを報告する。)
  • パターンの絞り込みができたときに、ルール1(all pattarn,no pattern)を適用する。
素朴な NiceLoop アルゴリズムは、盤面全体のセル・候補数字を起点とする探索であり、試行回数は多い。
一方、POMパターンを起点とする探索は パターン内に矛盾検出が保証されるので、効率が良い。
なお、ルール2でパターンを絞った状態で適用するのが良いでしょう。

 Pattern Overlay Method Rule4 - POM NiceLoop +

パターン数が 3以上のときは、1つのパターンのみ全要素が真であり、その他のパターンは真と偽が混在する。 アルゴリズムは次のようになる。

  • n パターンの場合、全てのパターンについて、NiceLoopの経路探索を行う。
  • n-1 パターンで矛盾が検出できた場合、残りの 1 パターンが真である。
  • NiceLoop は完全ではない(注)ので、n-1 パターンの矛盾を検出できないことはある。この場合は、真のパターンを決定できない。
    (注)完全ではない：NiceLoopに用いるリンクには様々な種類があるが、現状は充分なリンク種を含むNiceLoopアルゴリズムにはなっていない。
経験則であるが、このアルゴリズムの解は ALS,AIC,ALS-XZなどの複雑なリンクが多く使われる。この影響もあるだろうが、計算量が多い。
POMパターンの数が多いことと、複雑なリンクが必要なことは、局面の複雑さの”別の現れ”かもしれない。
