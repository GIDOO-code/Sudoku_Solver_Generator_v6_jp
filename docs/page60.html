 Multi Secter Locked Sets(MSLS)

Multi Secter Locked Sets(MSLS)は、複数のセクター(行、列、ブロック)にまたがるロックを用いるアルゴリズムです。 MSLSは、 Locked Set , Fish などの Locked 系アルゴリズムの拡張です。

Fishではアルゴリズムの説明に BaseSet / CoverSet を用いますが、ここでは以下の MSLS に合わせて、Secter / Linkを用います。




MSLS は次のようなアルゴリズムです。

  • 盤面にいくつかの Secter を配置し、未確定数字の交差部に着目します。 Secterが3重に重なる交差部は対象外とします。

  • 交差部にある候補数字に着目し、それらを結ぶリンクを求めます。候補数字を結ぶリンク表現が複数種あることもあります。
    1つの候補数字のリンク群について、これらをカバーする最小のリンク群を求めます。このときの リンク数 をカウントします。

  • 交差部のセル数と、合計リンク数 が同数であれば交差部は Locked Set になります。 次の例では 5セル⁼5リンクです。

  • リンクと同じSecterの 交差部以外の候補数字 は、除外されます。

 Multi Secter Locked Sets(MSLS) の証明

MSLS は次のように証明されます。

  • 交差部の セル数 と 合計リンク数 が同数であることから、候補数字ごとの 最小リンク数 は確定する。

  • 従って、候補数字ごとカバーするリンク群が決まりる。
    ただし、複数種のカバーパターンがあることもある (例えば Fish形のカバーリンク群)。

  • 従って、リンク数分の候補数字が肯定的に確定する。 |交差部Freeセル| = |リンク| から 交差部のセルは全て肯定的に確定する。
    ただし、"交差部の候補数字が肯定的に確定する"ことが決まるのであり、”候補数字が一意に決まる”のではない。




 SK Loop

SK Loopは、8次のMSLSです。2つの行セクター、2つの列セクター、4つのブロックセクター構成です。
セクターの配置は、次の図のようになります。
SK Loopではセクターの3重の重なりがあります。ここは交差部外に扱い、候補数字があれば除外できます。
SK LoopはMSLSなので、解法アルゴリズムに特殊性はありません。Secter集合を 2行2列4ブロック と指定することで、高速化が図れます。
